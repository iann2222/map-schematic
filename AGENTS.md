# AGENTS.md

## 專案概述

本專案為一套地圖示意圖製作工具。

核心原則：

- 執行時必須可離線運作
- 不得依賴任何外部 API 或線上地圖服務
- 地圖與地名資料必須來自官方資料包
- 僅在「首次初始化或更新資料包」時允許使用網路下載資料
- 不得在使用過程中呼叫任何第三方即時服務

---

## 資料政策（Data Policy）

1. 所有地圖底圖資料與地名資料必須來自官方資料包。
2. 官方資料包透過 GitHub Releases 發佈。
3. 資料包下載僅允許在：
   - 首次初始化
   - 使用者主動更新資料時
4. 使用過程中不得向外部伺服器請求地理資料或地圖瓦片。
5. 大型資料檔不得直接 commit 至 Git repository。
6. 必須支援資料包版本檢查與相容性驗證。
7. 更新資料包時需先下載完整檔案並驗證後才切換版本。

---

## 架構原則（Architecture Principles）

1. 採模組化設計：
   - 資料層（Data Layer）
   - 地圖渲染層（Rendering Layer）
   - 標示系統（Annotation Engine）
   - 專案檔系統（Project System）
   - 資料包管理系統（Data Pack Manager）

2. 地圖尺度限制：
   - 僅支援中尺度地圖
   - 不支援街道、建築或地址層級
   - 必須限制最大縮放層級

3. 所有資料路徑必須透過統一設定取得，不可硬編碼絕對路徑。

4. 必須明確處理錯誤狀況：
   - 資料包缺失
   - 資料版本不相容
   - 下載失敗
   - 檔案損壞

---

## 編碼規範（Coding Rules）

1. 不得新增任何會導致執行時依賴外部 API 的功能。
2. 不得在程式中直接寫死下載網址，必須透過版本管理設定讀取。
3. 所有資料讀取行為需集中於資料層，不得散落於 UI 邏輯中。
4. 所有新增功能需：
   - 明確區分線上初始化流程與離線運行流程
   - 確保離線模式下可正常運作
5. 必須避免引入不必要的第三方依賴。

---

## 專案檔規範

1. 專案檔格式為 `.mapproj`
2. 專案檔需包含：
   - schemaVersion
   - dataPackVersion
3. 若專案使用之資料版本與本機資料版本不一致，需提示使用者處理。

---

## 未來擴充原則

1. 若新增資料來源，必須可整合至官方資料包。
2. 不得引入即時線上地圖瓦片服務。
3. 必須維持可完全離線運行之特性。
